# 08a スコアシステム - 算出ロジック

## 概要

スコアアタック形式。クリアまたはゲームオーバー時に総合スコアを算出する。

---

## 基本スコア

| 項目 | 計算式 | 説明 |
|------|--------|------|
| 撃破スコア | 敵ごとの経験値 x 10 | 撃破した全敵の合計 |
| フロアスコア | クリアフロア数 x 500 | 到達フロアまで加算 |
| ボスボーナス | ボス撃破数 x 3000 | ボスを倒すごとに加算 |

---

## ボーナススコア

| 項目 | 計算式 | 説明 |
|------|--------|------|
| 知識ボーナス | 獲得知識数 x 200 | 集めた知識の数 |
| コンボボーナス | 連続ぴったり撃破数 x 100 x コンボ数 | 連続で0撃破した回数（途中で失敗するとリセット） |
| 残HPボーナス | 残HP x 50 | クリア時のみ適用 |
| 残MPボーナス | 残MP x 30 | クリア時のみ適用 |

---

## ペナルティ

| 項目 | 計算式 | 説明 |
|------|--------|------|
| ターンペナルティ | -総ターン数 x 2 | ターン数が多いほど減点 |
| 幽霊化ペナルティ | -幽霊化させた回数 x 50 | 敵を幽霊化させるごとに減点 |

---

## 最終スコア

```
最終スコア = 基本スコア + ボーナススコア + ペナルティ（最低0）
```

スコアは0を下回らない。

---

## コンボシステム

- 敵をぴったり0で撃破するとコンボカウンタが+1
- 幽霊化させる（負にしてしまう）とコンボリセット
- 技を外す（敵がいない方向に使う）ではリセットしない
- コンボ数はHUDに表示

### コンボボーナス計算例

```
1コンボ目: 100 x 1 = 100
2コンボ目: 100 x 2 = 200
3コンボ目: 100 x 3 = 300
...
10コンボ目: 100 x 10 = 1000

合計（10コンボ）: 100 x (1+2+...+10) = 5500
```

---

## 受け入れ条件

| ID | 条件 |
|----|------|
| AC-SCR-001 | 敵撃破時にスコアが加算される |
| AC-SCR-002 | コンボカウンタが正しく増加・リセットされる |
| AC-SCR-003 | ゲーム終了時に全項目のスコアが正しく算出される |
| AC-SCR-006 | スコアが0を下回らない |
