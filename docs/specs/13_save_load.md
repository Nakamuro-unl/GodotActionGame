# 13 セーブ・ロード仕様書

## 概要

インゲーム中の状態を1スロットのセーブデータとしてローカルに保存し、タイトル画面から「つづきから」で復元する。

---

## セーブデータ

### 保存先

`user://savegame.json`（1スロットのみ）

### 保存タイミング

- フロア遷移時（階段使用時）に自動セーブ
- メニューからの手動セーブ（将来対応）

### 削除タイミング

- ゲームクリア時に自動削除（パーマデス）
- ゲームオーバー時に自動削除（パーマデス）

## セーブフロー

```
1. フロア遷移（階段使用）をトリガーとする
2. 新フロア生成後の状態をシリアライズ
3. user://savegame.json に書き出し
4. 「セーブしました」メッセージ表示
```

---

## ロードフロー

```
1. タイトル画面で「つづきから」を選択
2. user://savegame.json を読み込み
3. GameSession の各システムに状態を復元:
   a. セッション情報（stage, floor, seed）
   b. マップグリッド復元
   c. プレイヤーステータス復元
   d. 敵の再配置
   e. 宝箱・ギミック復元
   f. 知識・スコア・ターン復元
4. InGameシーンに遷移
```

---

## タイトル画面の変更

```
> はじめから
  つづきから    ← セーブデータがあれば選択可能
  ランキング
  あそびかた
  せってい
```

- セーブデータが存在しない場合「つづきから」はグレーアウト
- 「はじめから」選択時、既存セーブデータがあれば上書き確認は不要（ローグライクのため）

---

## パーマデスとの整合

- ゲームオーバー時: セーブデータ削除
- ゲームクリア時: セーブデータ削除
- 途中でアプリを閉じても、最後のフロア遷移時点から再開可能

---

## 受け入れ条件

| ID | 条件 |
|----|------|
| AC-SAVE-001 | フロア遷移時にセーブデータが保存される |
| AC-SAVE-002 | セーブデータからゲーム状態を完全に復元できる |
| AC-SAVE-003 | プレイヤーのHP/MP/Lv/技/アイテムが復元される |
| AC-SAVE-004 | 敵の位置・数値・状態が復元される |
| AC-SAVE-005 | マップのグリッドが復元される |
| AC-SAVE-006 | 知識・スコア・ターン数が復元される |
| AC-SAVE-007 | セーブデータがない場合「つづきから」が選択不可 |
| AC-SAVE-008 | ゲームオーバー/クリア時にセーブデータが削除される |
| AC-SAVE-009 | セーブデータのバージョン不一致時にロード拒否する |
