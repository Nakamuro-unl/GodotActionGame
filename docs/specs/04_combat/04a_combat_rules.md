# 04a 戦闘システム - 基本ルール

## 概要

本ゲームのコアメカニクス。敵の頭上に表示された数値をぴったり0にすることで撃破する。数学の知識を技として使い、戦略的に数値を操作する。

---

## 基本ルール

1. 敵はそれぞれ固有の「数値」を持つ（頭上に常時表示）
2. プレイヤーは隣接時に技を使い、敵の数値を変化させる
3. 数値が **ぴったり0** になると撃破
4. 数値が **負** になると幽霊化
5. 幽霊化した敵も再度0にしなければ撃破できない

---

## 数値の性質

| 状態 | 数値の範囲 | 敵の状態 | 備考 |
|------|-----------|---------|------|
| 通常 | 1以上の整数 | 実体あり | 通常攻撃・技が有効 |
| ゼロ | 0 | 撃破 | 経験値・スコア獲得 |
| 負数 | -1以下の整数 | 幽霊化 | 物理接触ダメージ無効、技のみ有効 |

### 幽霊化の詳細

- 数値が負になった瞬間に幽霊化する
- 幽霊状態の敵は半透明で表示
- 幽霊は壁を通過できる
- 幽霊はプレイヤーに接触してもダメージを与えない
- 幽霊に技を使って数値を0に戻すと撃破できる
- 幽霊は毎ターン数値が+1される（自然回復で実体に戻ろうとする）

---

## ダメージ計算（敵→プレイヤー）

敵がプレイヤーに隣接している場合、敵ターンに攻撃する。

```
ダメージ = 敵の攻撃力 - プレイヤーの防御力（最低1）
```

- 敵の攻撃力はステージ・種類によって異なる
- プレイヤーの防御力は初期0、装備・レベルで上昇

---

## 戦闘の流れ（1ターン）

```
1. プレイヤーフェーズ
   a. 移動 or 技使用 or アイテム使用 or 待機
   b. 技使用時: 方向指定 → 対象の敵の数値に効果適用
   c. 数値が0 → 敵撃破（経験値・スコア獲得）
   d. 数値が負 → 敵幽霊化

2. 敵フェーズ
   a. 全敵が順番に行動（移動 or 攻撃）
   b. 幽霊の自然回復（数値+1）
   c. 幽霊の数値が0になった場合 → 撃破（自滅扱い、スコア低め）
```

## 受け入れ条件

| ID | 条件 |
|----|------|
| AC-CMB-001 | 敵の頭上に数値が常時表示される |
| AC-CMB-002 | 技の効果が敵の数値に正しく適用される |
| AC-CMB-003 | 数値が0になった敵が撃破される |
| AC-CMB-004 | 数値が負になった敵が幽霊化する |
| AC-CMB-005 | 幽霊化した敵が半透明で表示される |
| AC-CMB-006 | 幽霊化した敵が壁を通過できる |
| AC-CMB-007 | 幽霊は毎ターン数値が+1される |
| AC-CMB-008 | 技の使用でMPが正しく消費される |
| AC-CMB-009 | 敵がプレイヤーに隣接時にダメージを与える |
| AC-CMB-010 | 撃破時に経験値とスコアが加算される |
