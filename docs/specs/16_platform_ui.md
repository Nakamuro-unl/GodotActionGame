# 16 プラットフォーム別UI仕様書

## 概要

PC（キーボード）とモバイル（タッチ）でHUD構成を切り替える。プラットフォーム判定はタッチスクリーンの有無で行う。

---

## プラットフォーム判定

```gdscript
var is_mobile: bool = DisplayServer.is_touchscreen_available()
```

---

## 画面構成比較

| 表示要素 | PC | モバイル |
|---------|-----|---------|
| バーチャルパッド | 非表示 | 表示（技アイコン付き） |
| キーボードヒント | 表示 | 非表示 |
| 技スロット（HUD下部） | アイコン+テキスト+キー番号 | 非表示（パッドに統合） |
| HUD上部 | HP/MP/Lv/Floor/Turn/向き | 同じ |
| ミニマップ | 右上 | 右上 |
| メッセージログ | 下部 | パッドの上 |
| デバッグ情報 | 右上 | 右上 |
| 「調べる」ボタン | Enterキーヒント | バーチャルパッドに統合 |
| メニューボタン | Escキーヒント | バーチャルパッドに追加 |

---

## PC表示

### 技スロット（画面下部）

```
[減法icon][1]マイナス5  [加法icon][2]プラス3  [3]---  [4]---  [5]---  [6]---
```

### キーボードヒント（画面最下部）

```
矢印:移動  Shift+矢印:転換  1-6:技  I:アイテム  Enter:調べる  Esc:メニュー  Space:待機
```

---

## モバイル表示

### バーチャルパッド（技アイコン付き）

```
                              [icon]技1  [icon]技2  [icon]技3
[  ^  ]                       [icon]技4  [icon]技5  [icon]技6
[< >]   [マップ]
[  v  ]                       [待機]  [調べる]  [menu]
[移動/転換]
```

- 技ボタンに対応する知識のアイコンを表示
- 「階段」ボタンを「調べる」に名称変更（階段/宝箱/ギミック統合）
- 「menu」ボタンでインベントリ開閉
- 「アイテム」ボタンを「menu」長押しで表示（将来対応）

---

## 受け入れ条件

| ID | 条件 |
|----|------|
| AC-PUI-001 | PCではバーチャルパッドが非表示になる |
| AC-PUI-002 | PCではキーボードヒントが表示される |
| AC-PUI-003 | モバイルではバーチャルパッドが表示される |
| AC-PUI-004 | モバイルでは技スロット行とキーヒントが非表示になる |
| AC-PUI-005 | モバイルのバーチャルパッド技ボタンにアイコンが表示される |
| AC-PUI-006 | モバイルに「調べる」「menu」ボタンがある |
