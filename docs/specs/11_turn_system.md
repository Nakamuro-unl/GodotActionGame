# 11 ターンシステム仕様書

## 概要

不思議のダンジョン型のターン制システム。プレイヤーが1アクションを行うと1ターンが進行し、その後全敵が行動する。

---

## ターンの流れ

```
1. プレイヤーフェーズ（入力待ち）
   → プレイヤーがアクションを実行
2. 敵フェーズ（自動）
   → 全敵が順番に行動
3. 環境フェーズ（自動）
   → 幽霊の自然回復、罠の効果、HP自然回復チェック等
4. ターンカウント +1
5. 1に戻る
```

---

## フェーズ定義

| フェーズ | 実行者 | 内容 |
|---------|--------|------|
| PLAYER_INPUT | システム | プレイヤーの入力を待つ |
| PLAYER_ACTION | プレイヤー | 移動/技使用/アイテム使用/待機 |
| ENEMY_ACTION | 各敵 | AI に基づく移動/攻撃 |
| ENVIRONMENT | システム | 幽霊回復、自然回復、状態異常処理 |
| TURN_END | システム | ターンカウント加算、次ターンへ |

---

## ターン消費ルール

| アクション | ターン消費 |
|-----------|-----------|
| 移動（上下左右） | 1 |
| 技の使用 | 1 |
| アイテム使用 | 1 |
| 待機 | 1 |
| メニューを開く | 0 |
| 足元を調べる | 0 |
| 移動失敗（壁） | 0 |
| 技キャンセル | 0 |

---

## TurnManager の責務

- 現在のフェーズを管理する
- フェーズ遷移を制御する
- ターンカウントを保持する
- 各フェーズ開始時にシグナルを発火する

### シグナル

| シグナル | 引数 | 発火タイミング |
|---------|------|--------------|
| turn_started | turn_number: int | 新しいターンの開始時 |
| phase_changed | phase: Phase | フェーズが変わった時 |
| player_phase_started | - | プレイヤー入力待ち開始時 |
| enemy_phase_started | - | 敵フェーズ開始時 |
| environment_phase_started | - | 環境フェーズ開始時 |
| turn_ended | turn_number: int | ターン終了時 |

---

## 受け入れ条件

| ID | 条件 |
|----|------|
| AC-TURN-001 | プレイヤーのアクション後にターンが1進む |
| AC-TURN-002 | ターン消費0のアクションではターンが進まない |
| AC-TURN-003 | プレイヤーフェーズ→敵フェーズ→環境フェーズの順で処理される |
| AC-TURN-004 | ターンカウントが正しく加算される |
| AC-TURN-005 | 各フェーズ開始時にシグナルが発火する |
| AC-TURN-006 | 環境フェーズで幽霊の自然回復が処理される |
| AC-TURN-007 | 環境フェーズで10ターンごとにHP+1の自然回復が処理される |
